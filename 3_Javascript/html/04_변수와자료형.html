변수 선언 방식

        JavaScript는 변수 선언 시 자료형이 아닌
        var, let, const, (미작성) 중 하나를 골라서 작성함

        var   : 변수, 변수명 중복 가능(덮어쓰기), 함수 레벨 scope
        let   : 변수, 변수명 중복 X, 블록{} 레벨 scope
        const : 상수, 변수명 중복 X, 블록{} 레벨 scope
        미작성 : var이랑 똑같음 / 근데 이게 여기서 선언한 변수인지 전역변수인지 헷갈리니까 미작성은 잘 쓰지 않는데

        1순위 : const
                JS는 HTML상에서 특정 요소를 선택해 지속적으로 추적하는 경우가 많아
                상수형 변수에 저장해 고정적으로 사용
                (요소 하나를 찾아서 계속 추적하게 만들어야 되는데 변수면 값이 변하면서 추적하기가 어렵겠지)
        2순위 : let
        3순위 : var
        미작성 : 미작성

JavaScript의 자료형

        JS는 변수 선언 시 별도의 자료형을 지정하지 않음.
        -> 변수에 대입되는 값(리터럴)에 의해서 자료형이 결정됨.

        - string (문자열 / "abc", 'abc')
        - number (숫자-(정수, 실수 구분 안함) / 100(정수), -3.154(실수))
        - boolean (논리 / true, false)
        - object (객체 / 배열(Array), JS객체 {K:V, K:V} )
        - function (함수)
        - undefined (자료형이 정의되지 않은 변수)
            -> 값이 대입되지 않은 변수

        * null은 참조하는게 없음을 나타내는 리터럴(자료형 X)
